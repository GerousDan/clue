Clue Solver

About this project

About this project: Clue (or Cluedo outside of North America) is a game published by Hasbro in which players make accusations to try to determine who killed Mr. Boddy. This application takes information about the game (including what cards you have and which suggestions are disproved by whom) to figure out who has which cards, and thus who the killer is.

To get a quick start, load one of these sample clue games and play around with the Game Info and Simulations:

  • 63-QLU.3-ANQIHOLUMG.3-QLU-AMG-ANH-AOI.3QLU-AFECSNBTIHKOGRPMJD.3-QLU.3-QLU.3-QLU.
  • 54TNJS-AFECBIHKOLURQPMGD.4-ANSTOJ-FHP.4-ANSTOKJP.3-FNSHTJP-AO-AK.3-FNSHTJP.3-TNJS. (the simulation for this one has lots of colors :-) )

The front-end was developed using the Google Web Toolkit, the back-end is written in Python, and they communicate in the JSON data format. This was my first project using the Google Web Toolkit, and I didn't spend a lot of time prettying things up, etc.

I originally tried to write the back-end using first-order logic (see discussions on my LiveJournal), and then propositional logic, but ended up rolling my own deduction, which was sort of disappointing.

The simulations are only reliable to within a few percent (it only does 2000 of them) - basically it chooses each possible solution, tries to assign cards to each player and sees if the solution is consistent or not (and if so, counts it).

The name suggestions are taken from popular names from the 1990s census - I took the top 750 for males and females.

The images on the Game Info tab are taken from the Silk icon set at famfamfam.com - thanks!

Here are my source files:

  • clueengine.py - the Python backend that does the deductions and simulations.
  • clue.py - the CGI script that handles communications between the frontend and cluesolver.py.
  • json.py - the Python code for writing JSON-formatted messages, taken from the sourceforge project json-py.
  • ClueSolver.java - the main frontend interface. (written to be compiled with GWT)
  • ClueStateWidget.java - the widget that holds what we know about each card and displays the appropriate image and tooltip.
  • NameSuggestPanel.java - the textbox for entering a name that suggests common names.
  • CgiHelper.java - A helper for calling CGI scripts.
  • CgiResponseHandler.java - an interface for what to do when getting a response from a CGI script.
  • ConfirmHandler.java - a simple interface for what to do when the user confirms an action.
